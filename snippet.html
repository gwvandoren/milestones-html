The Anatomy of an HTMl Tag

This document describes the following snippet:

<p>

Want to search the web?

Try <a href="https://google.com" id="the-best-link" class="banana" target="_blank">Google</a>!

</p>

The tags in the snippet are: the <p> tag and the <a> tag.

This link describes the <p> tag.

This link describes the <a> tag.

In terms of nesting, the <p> tag is a child of the <a> tag.

The <p> and <a> tags are the opening tags.

The </p> and </a> tags are the closing tags.

The attribute names on the <a> tag are:

  • href
    • The URL that the hyperlink points to. Links are not restricted to HTTP-based URLs — they can use any URL scheme supported by browsers:
      • Sections of a page with fragment URLs
      • Pieces of media files with media fragments
      • Telephone numbers with tel: URLs
      • Email addresses with malito: URLs
      • While web browsers may not support other URL schemes, web sites can with registerProtocolHandler()
  • id
    • Defines a unique identifier (ID) which must be unique in the whole document. Its purpose is to identify the element when linking (using a fragment identifier), scripting, or styling (with CSS).
  • class
    • A space-separated list of the classes of the element. Classes allows CSS and JavaScript to select and access specific elements via the class selectors or functions like the method Document.getElementsByClassName().
  • target
    • Where to display the linked URL, as the name for a browsing context (a tab, window, or <iframe>). The following keywords have special meanings for where to load the URL:
      • _self: the current browsing context. (Default)
      • _blank: usually a new tab, but users can configure browsers to open a new window instead.
      • _parent: the parent browsing context of the current one. If no parent, behaves as _self:.
      • _top: the topmost browsing context (the "highest" context that’s an ancestor of the current one). If no ancestors, behaves as _self:.
